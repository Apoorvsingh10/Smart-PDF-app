## PART 1 — SUBSCRIPTION PLANS

These are the exact plans to implement, hardcoded:

| Plan ID      | Label               | Price  | Duration        | AI Requests/month |
|--------------|---------------------|--------|-----------------|-------------------|
| free_trial   | Free Trial          | ₹0     | 2 requests ever | 2 total lifetime  |
| monthly      | Monthly             | ₹100   | 30 days         | 15/month          |
| quarterly    | Quarterly           | ₹250   | 90 days         | 15/month          |
| lifetime     | Early Bird Lifetime | ₹1,000 | Forever         | 15/month          |

Rules:
- Free trial: 2 AI requests total, never resets, no expiry
- Paid plans: 15 AI requests per month, resets on billing date
- After trial exhausted → show paywall
- After monthly/quarterly expires → show paywall
- Lifetime never expires
- If user hits 15/month limit → show "limit reached, resets on [date]" message

## PART 3 — AI FEATURES

### 3a. src/utils/AIManager.h and AIManager.cpp

QObject registered as QML singleton. Responsibilities:
- Read subscription from SubscriptionManager before every AI call
- If not premium → emit accessDenied() signal
- If limit reached → emit limitReached(resetDate) signal
- Store Claude API key from QSettings key "ai/apiKey"
- Q_PROPERTY QString currentPdfText
- Q_PROPERTY bool isImageBased
- Q_PROPERTY QStringList currentPageImages (base64 PNG pages)

Q_INVOKABLE methods:
- void summarizePdf()
- void askQuestion(const QString &question)
- void cancelRequest()

Signals:
- void responseReady(const QString &response)
- void loadingChanged(bool isLoading)
- void errorOccurred(const QString &error)
- void accessDenied()
- void limitReached(const QString &resetDate)

Claude API details:
- Endpoint: https://api.anthropic.com/v1/messages
- Model: claude-sonnet-4-5 for text PDFs
- Model: claude-sonnet-4-5 for image PDFs (also supports vision)
- Headers: Content-Type: application/json, x-api-key, anthropic-version: 2023-06-01
- max_tokens: 1024
- System prompt summarize: "You are a PDF assistant. Provide a clear structured summary 
  with key points, main topics, and important details."
- System prompt Q&A: "You are a PDF assistant. Answer based only on the PDF content. 
  Be concise. If answer not in PDF, say so."
- Text PDFs: send as text in content
- Image PDFs: send each page as base64 image block + text question block
- Cap text at 50,000 chars, warn if truncated
- Cap image pages at 10, warn if more pages exist
- After successful response → call SubscriptionManager.incrementUsage()

### 3b. src/utils/PDFTextExtractor.h and PDFTextExtractor.cpp

- Load PDF via QPdfDocument
- Extract all text page by page
- If total text < 50 chars → set isImageBased = true
- If isImageBased → render pages to QImage via QPdfDocument::render()
- Render at QSize(pageWidth * 2, pageHeight * 2) for quality
- Convert each page QImage to base64 PNG
- Return struct: { QString text, bool isImageBased, QStringList base64Pages }
- Expose as Q_INVOKABLE

## PART 4 — SUBSCRIPTION MANAGER

### src/utils/SubscriptionManager.h and SubscriptionManager.cpp

QObject singleton registered to QML. Responsibilities:
- On app start: fetch subscription doc from Firestore for current uid
- If doc doesn't exist: create default free_trial doc
- Cache subscription state locally
- Check if plan is expired on fetch

Q_PROPERTY (all read-only, notify signals):
- QString plan
- QString status  
- bool isPremium  (true if active monthly/quarterly/lifetime)
- bool isFreeTrial
- int aiRequestsUsed
- int aiRequestsLimit
- QString expiresAt (human readable)
- QString resetDate (human readable)

Q_INVOKABLE methods:
- void fetchSubscription() — load from Firestore
- bool canMakeAIRequest() — check limit + status
- void incrementUsage() — increment aiRequestsUsed in Firestore
- void applyPurchase(QString plan, QString orderId, QString paymentId) — update Firestore after payment

Firestore calls via QNetworkAccessManager using Firebase REST API:
- GET: https://firestore.googleapis.com/v1/projects/{PROJECT_ID}/databases/(default)/documents/users/{uid}/subscription
- PATCH: same URL for updates
- Use Firebase Auth ID token from AuthManager for Bearer auth header

## PART 5 — RAZORPAY PAYMENT INTEGRATION

### android/src/main/java/io/smartpdf/app/RazorpayHelper.java

New Java file that:
- Imports Razorpay Android SDK
- Has method startPayment(String orderId, int amountPaise, String description, String email)
- Implements RazorpayPaymentResultListener
- On success: calls back to Qt via JNI method 
  io.smartpdf.app.SmartPdfActivity.onPaymentSuccess(String paymentId, String orderId)
- On failure: calls back 
  io.smartpdf.app.SmartPdfActivity.onPaymentFailed(String errorCode, String errorDesc)

### Update android/src/main/java/io/smartpdf/app/SmartPdfActivity.java

Add:
- static native void onPaymentSuccess(String paymentId, String orderId)
- static native void onPaymentFailed(String errorCode, String errorDesc)
- Method to launch RazorpayHelper.startPayment()
- Called from Qt C++ via QAndroidJniObject

### android/build.gradle

Add Razorpay dependency:
implementation 'com.razorpay:checkout:1.6.40'

### src/utils/PaymentManager.h and PaymentManager.cpp

QObject singleton registered to QML:

Q_INVOKABLE void startPayment(const QString &planId)
- planId: "monthly" | "quarterly" | "lifetime"
- Map to amount: monthly=10000 paise, quarterly=25000, lifetime=100000
- Generate a local orderId (UUID) — in production this comes from your backend
- Call Android JNI to launch Razorpay checkout

Signals:
- void paymentSuccess(const QString &paymentId, const QString &orderId, const QString &plan)
- void paymentFailed(const QString &errorCode, const QString &errorDesc)

On paymentSuccess signal:
- Call SubscriptionManager.applyPurchase(plan, orderId, paymentId)
- Emit purchaseComplete() for QML to react

Razorpay config to use (hardcode these):
- Key ID: read from QSettings key "razorpay/keyId"
- Business name: "Smart PDF"
- Description: plan label
- Theme color: use Theme.primaryColor value (#6200EE or whatever Theme.qml defines)
- Prefill email: from AuthManager current user email

Note: For MVP, skip backend order creation. Use client-side flow with Razorpay key only.
In production later, add Firebase Cloud Function to create Razorpay orders server-side.

## Before Running This Prompt

Two things to set up first:

**Razorpay account** — go to razorpay.com, sign up with your PAN + bank account, get your Key ID from the dashboard. Takes 1–2 days for KYC approval.

**QSettings initialization** — once you have keys, add this once to your app on first run or in SettingsScreen:
```cpp
QSettings settings;
settings.setValue("ai/apiKey", "sk-ant-...");
settings.setValue("razorpay/keyId", "rzp_live_...");
settings.setValue("firebase/projectId", "your-project-id");
```
